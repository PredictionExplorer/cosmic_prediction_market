# Gesture Market — frontend

A Next.js app for trading the [Gesture Market](../README.md): a scalar prediction
market on how many gestures (bids) the current Cosmic Signature round will end
with, settled in CST on Arbitrum One.

Everything a trader needs lives on one screen: the live crowd prediction with
its history, one-click HIGHER/LOWER bets with exact client-side quotes, your
position marked to the live gesture count with an interactive "what if the
round ends at N" explorer, one-click resolution when the round ends, and
one-click claiming after resolution.

## Stack

- [Next.js 16](https://nextjs.org) (App Router, Turbopack) + React 19 + TypeScript (strict)
- [Tailwind CSS v4](https://tailwindcss.com) with a custom cosmic design system
- [wagmi v3](https://wagmi.sh) + [viem](https://viem.sh) for wallets and contract I/O
  (injected wallets are auto-discovered via EIP-6963; WalletConnect optional)
- [TanStack Query](https://tanstack.com/query) for on-chain data caching/refresh
- [Vitest](https://vitest.dev) + Testing Library + [fast-check](https://fast-check.dev)
  property tests (250+ assertions, mirroring the Solidity fuzz suite)
- [motion](https://motion.dev) for tasteful animation, [sonner](https://sonner.emilkowal.ski) for tx toasts

## Architecture

```
src/
  lib/          pure, framework-free logic (fully unit-tested)
    math.ts       exact bigint mirror of GestureMarket.sol (quotes, payouts,
                  break-evens, slippage) — validated property-by-property
                  against the Foundry fuzz suite
    history.ts    rebuilds the full prediction chart from Bet events alone
                  (unwind current reserves backwards, replay forwards —
                  no archive node needed)
    market.ts     snapshot types, market phase, position valuation
    config.ts     env parsing/validation, ?market= override
    format.ts     display formatting + strict amount parsing
    errors.ts     every contract error mapped to a human explanation
    wagmi.ts      chain + connector config
    abi/          contract ABIs (generated by forge, checked in)
  hooks/        thin wagmi/react-query glue
    use-market.ts         one multicall for the full market+user snapshot
    use-market-events.ts  event scan + live watcher for chart & activity
    use-market-actions.ts approve/bet/mint/redeem/resolve/claim with a
                          uniform submit→confirm→refresh toast flow
  components/   presentational components (props in, callbacks out)
```

The design rule: **all market math happens in `lib/` with bigints, identically
to the contract** (floor/ceil rounding included). Components never compute
prices themselves — they render what the lib computes, and the lib is what the
tests hammer. Quotes are instant and RPC-free; the slippage guard
(`minTokensOut`) is computed from the same math at submit time.

### Local sandbox (no real funds)

Run a disposable market against [anvil](https://getfoundry.sh) with mock CST
and a mock game you control:

```bash
# terminal 1 — local chain
anvil

# terminal 2 — deploy mock CST + game + market, seed balances
forge script script/DeployLocal.s.sol --rpc-url http://127.0.0.1:8545 \
  --private-key 0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80 \
  --broadcast
```

Then point the frontend at it in `frontend/.env.local`:

```bash
NEXT_PUBLIC_CHAIN_ID=31337
NEXT_PUBLIC_MARKET_ADDRESS=0x8A791620dd6260079BF849Dc5567aDC3F2FdC318
NEXT_PUBLIC_RPC_URL=http://127.0.0.1:8545
```

On the anvil chain the wallet modal offers a **Mock Connector** for anvil's
unlocked default accounts — no browser extension needed. Drive the game from
the command line:

```bash
GAME=0xe7f1725E7734CE288F8367e1Bb143E90bb3F0512
# more gestures come in
cast send $GAME "setNumBids(uint256,uint256)" 3 900 --private-key $PK --rpc-url http://127.0.0.1:8545
# end the round (unlocks resolve())
cast send $GAME "setRoundNum(uint256)" 4 --private-key $PK --rpc-url http://127.0.0.1:8545
```

## Configuration

All config is env-driven and validated at startup (`src/lib/config.ts`):

| Variable | Required | Purpose |
|---|---|---|
| `NEXT_PUBLIC_CHAIN_ID` | no (default 42161) | 42161 Arbitrum One, 31337 anvil |
| `NEXT_PUBLIC_MARKET_ADDRESS` | no | default market; `?market=0x…` in the URL overrides it |
| `NEXT_PUBLIC_RPC_URL` | no | custom RPC (falls back to the chain's public one) |
| `NEXT_PUBLIC_DEPLOY_BLOCK` | no | market's deploy block — speeds up event scans |
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` | no | enables WalletConnect for mobile wallets |

One deployment serves any number of per-round markets via the `?market=`
query parameter.

## Deploy to Vercel

1. Import the repository in Vercel and set **Root Directory** to `frontend`
   (framework preset: Next.js — auto-detected).
2. Add the `NEXT_PUBLIC_*` environment variables above.
3. Ship. Each new round's market is just an env var change (or share links
   with `?market=`).

## Testing philosophy

- `lib/math` carries the same economic properties the Solidity fuzz suite
  enforces (pool never loses, quotes equal execution, path independence within
  rounding, complete sets always worth 1 CST, payout monotonicity), so a quote
  shown in the UI is provably the number the contract will honor.
- `lib/history` is verified by roundtrip: any random bet sequence replayed
  forwards equals the chain state, and unwinding backwards recovers the
  opening pool exactly.
- Components are tested through user behavior (typing, clicking, sliding) with
  Testing Library — including validation, approval routing, slippage handling,
  and every market phase.
